# Data source

**Dataset:** Flaredown Autoimmune Symptom Tracker
**Source:** Kaggle — https://www.kaggle.com/datasets/flaredown/flaredown-autoimmune-symptom-tracker?resource=download
**Provider:** Flaredown (patient symptom-tracking app)
**File used:** `export.csv` — 686 MB, 7,976,223 rows, 42,283 users, 2012-05-18 → 2019-12-06
**Format:** long / event log — one row per `(user, date, trackable item)`
**License:** see the Kaggle dataset page for current license & terms before redistribution.

## Version control of the raw file (DVC)

The raw `export.csv` (686 MB) is **tracked by DVC, not git**. Git stores only the
small pointer `export.csv.dvc`; the bytes live in a DVC remote.

```
export.csv         md5  0a99224be0aabaade6996f4a96764766   size 686,174,513 bytes
```

Set up / use:
```bash
# 1. point the 'storage' remote at real storage (local dir, S3, GDrive, ...)
dvc remote modify storage url /abs/path/to/dvc-storage      # or s3://bucket/key
# 2. push the raw file into the remote (run once, on the machine that has it)
dvc add export.csv && dvc push
# 3. on a fresh clone: pull the raw file back
git clone <repo> && cd regression_health && dvc pull
```

The default remote in `.dvc/config` is a **placeholder** (`/path/to/your/dvc-storage`) —
repoint it before `dvc push`.

## Notes
- Treat `export.csv` as immutable; all derived data lives in `data/interim/` and
  `data/processed/` (also git-ignored, regenerated by `src/`).
- `.gitignore` excludes the raw + derived data; commit `export.csv.dvc` and the
  `.dvc/` directory so collaborators can `dvc pull`.
